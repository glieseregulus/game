  2. 移行して使う場合

    このアセットをインストールした後に移行したい場合はこちらEpic公式を参考にしてみてください。
    here
    このアセットはできるだけ移行しやすいように作ってはありますがアセットの全てを移行しない場合は特性上、アニメーションブループリントや構造体などは移行しづらいかもしれません。CSVは別でエクスポートしておくとエラーを出さずに移行できることが多いです。特にアニメーションブループリントの'F_Conversation'構造体変数は移行するとエラーを発生しやすいので作り直す等の対応が必要かもしれません。

    3. このアセットを使用する前のCSVの準備

    あなたがこのアセットを使いやすいと感じられるように考えて私は設計しました。
    その中でも、こういう用途はどうでしょう？
    あなたが使いたいストーリーのCSVさえ作っておけば、異なるマップでも'BP_TalkCharacter'をアウトライナーに配置し、ゲームモードを設定するだけでデモマップのように使うことができます。
    念の為に私が作った空のCSVテンプレートをこちらに置いておきますので使いたい人はダウンロードして使ってください。

    • CharacterName, TextBodyについて

      まずは上のセクションでダウンロードしたプレーンのCSVファイルで解説します。

      画像のCSVファイルのように今回は適当に打ち込んでいきます。
      まず、名前とテキストは好きに書いてください。
      あまり解説はいらないかもしれないです。
      名前に関してです。
      空欄になっている場合はテキストボックスごと消えるようになっています。
      その場合ログも残りません。
      テキスト本文に関しても特に言うことはありません。
      Unreal Engineやスプレッドシートで書いている場合、本文の改行が必要なときは大抵Alt+EnterかCtrl+Enterで改行することができます。
      なので、このことは覚えておきましょう。

    • Animation

      Animationの場所にはアニメーションモンタージュの参照パスを入れるようにしてください。
      下の画像を参考にしてください。


      この時、選択するべきアニメーションはアニメーションシーケンスではなくアニメーションモンタージュにすることと、Copy File PathではなくCopy Referenceを選択することを注意してください。
      今回はアセットに入れてあるサンプルアニメーションを順番に入れてみました。

    • SpawnCamera

      SpawnCameraはBoolean型ですのでTRUEかFALSEかでの記入が必要です。
      一応使わない時でも間違いやバグのないように空欄ではなくFALSEと記入しておきましょう。
      TRUEの場合、必ず次の項目のTargetLocationは記入しましょう。
      もしもTRUEに設定した時にTargetLocationを記入しなかった場合、ワールドのLocationRotation0でScale1でカメラが原点に切り替わる設定になってしまいます。注意してください。

      また、TextBodyがターゲットのキャラクターとの最後の会話の場合は必ずFALSEに設定してください。
      そうしなければTRUEになった最後のカメラ位置でそれが固定されて戻れなくなります。
      気をつけましょう。

    • TargetLocation

      SpawnCameraでも書いたように、SpawnCameraをTRUEにしたら必ず記入してください。
      念の為にFALSEの時は原点を記入しておきましょう。

      原点は
      (Rotation=(X=0.000000,Y=0.000000,Z=0.000000,W=1.000000),
      Translation=(X=0.000000,Y=0.000000,Z=0.000000),
      Scale3D=(X=1.000000,Y=1.000000,Z=1.000000))
      です。

      Scale3Dは必ず1にしましょう。
      他のものを設定する意味がないですし、よくわからなくなります。

      設定する位置はアナログな方法になってしまいますが、ビューポート左のPlaceActorsからCameraActorを選びワールドアウトライナーかビューポートにドロップしましょう。
      カメラの位置や角度をビューポート右下のワイプを見ながらちょうど良くしましょう。

      そして完成したトランスフォームの位置を確認して完了です。
      この動作はトランスフォームの位置確認をするための行動なので追加したカメラは必要ないので削除しておきましょう。
      ほんの少しのテクニックとして、このトランスフォームの入力はUnreal EngineにCSVをインポートした後に入力したほうが楽です。
      一度全部原点で設定しておいてから後で設定するとプロジェクトを作りやすいです。
      参考にしてみてください。

    • TargetTime

      この値はサードパーソンカメラからTargetLocationにカメラが切り替わるまでの時間を入れてください。
      この値はFloat変数での記入です。
      数字で記入しましょう。
      使わない場合は0を入れておくと良いです。

    Viewport manipulation during Setting Mode



    During the setting operation, you can rotate and inspect the mesh from the right half of the viewport.

    This operation can be performed by long-pressing and dragging the left mouse button. It automatically returns when closed with 'SaveSettings'.

    Information about the keyboard added to the save system and templates.

    This asset includes a simple save system.
    It is separate from the keyboard you have set in the key bindings.
    It has been set up for your immediate use, for now.
    The keyboard settings are in the image below.

    I will also briefly explain the save system.
    In the initial setup,
    pressing 'Save Settings' in the 'WG_Settings' widget returns you to the game and saves the settings.
    Similarly, when you press the 'Exit Game' button, it also saves the settings.
    As for 'Load', settings are not configured for keys other than the 2 key on the keyboard.
    However, there's a variable called 'Auto Load' in 'WG_Option'.
    If you want to autoload at the start of the game, set this to True.
    It's inputted from 'Event Construct' in 'WG_Option'.
    Also, a save data is created from 'WG_Option' Event Construct only during the game's first launch.
    If you don't need this during development, you might consider deleting it.
    Alternatively, you could manually incorporate it into the third key on the keyboard,
    giving you the option to handle it manually.
